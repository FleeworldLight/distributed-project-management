## 五、部署使用流程

### 5.1 启动后端（GitHub Codespace）

1. 打开仓库 → 点击 **Code** → **Create codespace on main**
2. Codespace 自动执行 `.devcontainer/setup.sh`，完成：
   - 安装/启动 MySQL + 创建数据库 `laigeoffer-pmhub` + 导入 `sql/init.sql`
   - 安装 Redis
   - Maven 编译（`mvn package -DskipTests`）
3. 按顺序开 8 个终端启动服务：

```bash
# 终端 1
mvn spring-boot:run -pl pmhub-eureka
# 终端 2
mvn spring-boot:run -pl pmhub-gateway
# 终端 3
mvn spring-boot:run -pl pmhub-auth
# 终端 4
mvn spring-boot:run -pl pmhub-modules/pmhub-system
# 终端 5
mvn spring-boot:run -pl pmhub-modules/pmhub-project
# 终端 6
mvn spring-boot:run -pl pmhub-modules/pmhub-workflow
# 终端 7
mvn spring-boot:run -pl pmhub-modules/pmhub-job
# 终端 8
mvn spring-boot:run -pl pmhub-monitor
```

4. Codespace **Ports** 面板找到 `6880`（Gateway），点小地球图标获取公网 URL，如：
   ```
   https://someone-6880.preview.app.github.dev
   ```

### 5.2 部署前端（GitHub Pages）

1. 仓库 **Settings → Secrets and variables → Actions** 添加密钥：
   - **`BACKEND_URL`** = 上一步的 Gateway 公网 URL
2. 推送 `main` 分支，Actions 自动构建 `pmhub-ui` 并部署到 Pages
3. 访问地址：`https://<org>.github.io/distributed-project-management/`

### 5.3 日常开发

| 操作 | 方法 |
|------|------|
| 改后端代码 | Codespace 中修改，重启对应服务 |
| 改前端代码 | 本地修改 → `git push` → Actions 自动重新部署 |
| 查看后端日志 | Codespace 对应终端 |
| 停止服务 | 每个终端 Ctrl+C |
