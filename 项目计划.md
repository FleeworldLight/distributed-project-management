
# DPM 分布式项目管理系统 - 项目计划

## 核心功能

| 模块 | 功能 |
|------|------|
| 系统管理 | 用户/角色/权限/菜单管理，RBAC 权限控制 |
| 项目管理 | 项目CRUD、成员管理、阶段划分 |
| 任务看板 | 拖拽式任务卡片（待办/进行中/已完成） |
| 工作流审批 | BPMN流程定义、审批节点流转、审批历史 |
| 数据看板 | 项目进度统计、任务分布图表 |
| 定时任务 | 任务调度、超时提醒 |

---

## 二、技术架构

```
前端 (Vue3 + Element Plus + ECharts) → GitHub Pages
                    ↕ API
Spring Cloud Gateway (6880) → 路由 + JWT鉴权
         ↕
Eureka 注册中心 (8761)
         ↕
认证(6800) ↔ 系统(6801) ↔ 项目(6806) ↔ 流程(6808) ↔ 定时(6803) ↔ 监控(6888)
         ↕               ↕                ↕               ↕
       Redis缓存      MySQL(系统库)     MySQL(项目库)    MySQL(流程库)
```

### 服务列表

| 服务 | 端口 | 说明 |
|------|------|------|
| pmhub-eureka | 8761 | Eureka 注册中心（新增） |
| pmhub-gateway | 6880 | Spring Cloud Gateway 网关 |
| pmhub-auth | 6800 | JWT 认证授权中心 |
| pmhub-system | 6801 | 用户/角色/权限管理 |
| pmhub-project | 6806 | 项目/任务/成员管理（核心） |
| pmhub-workflow | 6808 | Flowable 工作流引擎（核心） |
| pmhub-job | 6803 | 定时任务调度 |
| pmhub-monitor | 6888 | Spring Boot Admin 监控中心 |
| pmhub-ui | — | Vue3 前端 |

### 已删除的模块

- `pmhub-boot`（单体版，与微服务版本无关）
- `pmhub-gen`（代码生成器，非核心业务）
- `pmhub-base-seata`（分布式事务，课设不启用）
- `pmhub-base-sensitive`（敏感词过滤）
- `pmhub-base-datasource`（多数据源组件）
- `pmhub-base-notice`（RocketMQ + 企微通知）

### 架构调整说明（与原项目对比）

| 组件 | 原来 | 现在 | 原因 |
|------|------|------|------|
| 注册中心 | Nacos | Eureka | 轻量级，零依赖，适合Render部署 |
| 配置中心 | Nacos Config | 本地 application.yml | Nacos从未启用，直接写入本地配置 |
| 限流/熔断 | Sentinel | 移除 | 课设不可演示，降低复杂度 |
| 消息队列 | RocketMQ | 移除 | 部署太重，后续可选Redis Pub/Sub替代 |
| 分布式事务 | Seata | 移除 | 课程场景无需跨服务事务 |
| 通知系统 | 企微Webhook | 移除 | 具体业务绑定，无通用价值 |

## 六、后续操作步骤

### 步骤1：数据库准备

创建 MySQL 数据库 `laigeoffer-pmhub`，找到原始项目 `pmhub-master/sql/` 目录下的初始化脚本执行：

```sql
CREATE DATABASE IF NOT EXISTS `laigeoffer-pmhub` DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

然后导入表结构和初始数据（`pmhub-master/sql/laigeoffer-pmhub.sql` 或类似名称）。

### 步骤2（方案A）：Docker 一键启动

适用于 Codespace / 云服务器，一键拉起所有服务：

```bash
mvn package -DskipTests -q          # 1. 编译所有 JAR
docker compose build                  # 2. 构建 Docker 镜像
docker compose up -d                  # 3. 启动
```

也可用 `.\build-and-run.ps1 rebuild` 一步完成。

| 服务 | 地址 |
|------|------|
| Eureka 注册中心 | http://localhost:8761 |
| Gateway 网关 | http://localhost:6880 |
| Monitor 监控 | http://localhost:6888 (admin/123456) |

### 步骤2（方案B）：本地手动启动

适用于 Windows 本机开发调试，需先准备好依赖：

**1. 启动基础设施**
```bash
# Redis（确保本地已安装）
redis-server

# MySQL — 创建数据库并导入表结构
mysql -u root -p123456 -e "CREATE DATABASE IF NOT EXISTS laigeoffer-pmhub DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
mysql -u root -p123456 laigeoffer-pmhub < sql/init.sql
```

**2. 按顺序启动服务（用 IDE 或命令行）**
```
pmhub-eureka (8761) → 等待注册中心就绪
pmhub-gateway (6880)
pmhub-auth (6800)
pmhub-system (6801)
pmhub-project (6806)
pmhub-workflow (6808)
pmhub-job (6803)
pmhub-monitor (6888)
```

各模块的 Application 类（`*Application.java`）可直接在 IntelliJ 中右键运行，方便调试。

### 步骤3：启动前端

```bash
cd pmhub-ui
npm install
npm run dev
```

访问 http://localhost:9527 确认登录页面正常显示（前端 dev server 会自动代理 API 请求到 Gateway 6880 端口）。

### 步骤4：替换Logo/图标

用团队自定义图标替换 `pmhub-ui/src/assets/logo` 下的图片文件。

### 步骤5：编写实验报告

按课程要求格式编写综合实验报告，内容涵盖：
- 问题陈述与目标（中小团队项目管理效率低下）
- 系统设计与技术方案（Spring Cloud微服务架构图）

- 系统演示（功能截图 + 部署架构）
- 结论与收获

### 步骤6：录制演示视频

本地启动全部服务，录制3-5分钟功能演示视频，展示：
1. 系统登录 + 验证码

## 七、部署方案

### 方案A：Docker 容器化部署（推荐）

适用于 Codespace / 云服务器 / 本地演示，全部服务在 Docker 网络内通信。

**文件结构：**

| 文件 | 说明 |
|------|------|
| `docker-compose.yml` | 编排 10 个容器（MySQL + Redis + 8 个微服务） |
| `sql/init.sql` | 首次启动自动初始化数据库 |
| `build-and-run.ps1` | 一键构建脚本 |
| 各模块 `Dockerfile` | 基于 `openjdk:8-jre-slim` 构建镜像 |

**环境变量覆盖（docker-compose 自动注入）：**

| 变量 | 作用 |
|------|------|
| `EUREKA_CLIENT_SERVICEURL_DEFAULTZONE` | 注册中心地址 → `http://eureka:8761/eureka/` |
| `SPRING_DATASOURCE_DRUID_MASTER_URL` | 数据库 JDBC URL → `mysql:3306` |
| `SPRING_REDIS_HOST` | Redis 主机 → `redis` |

**命令速查：**

```bash
.\build-and-run.ps1 rebuild   # 完整流程
docker compose logs -f         # 查看实时日志
docker compose ps              # 查看容器状态
docker compose down            # 停止并删除
```

### 方案B：GitHub Pages（前端）

```
pmhub-ui/  →  npm run build:prod  →  dist/  →  GitHub Pages
```

### 方案C：本地开发模式

本地分别启动 Redis、MySQL，然后用 IDE 逐个运行服务，便于调试。
